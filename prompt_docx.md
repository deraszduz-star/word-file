# Промт для создания Word-документа (.docx) из Markdown-текста ВКР

## Общая задача

Создай Python-скрипт, который конвертирует Markdown-файл с текстом ВКР (выпускной квалификационной работы) в файл формата `.docx` с соблюдением всех требований оформления, перечисленных ниже.

---

## 1. Параметры страницы

- Формат: A4 (210 × 297 мм)
- Поля: левое — 30 мм, правое — 10 мм, верхнее — 20 мм, нижнее — 20 мм
- Ориентация: книжная

---

## 2. Основной текст

- Шрифт: Times New Roman, 14 пт, чёрный цвет
- Межстрочный интервал: полуторный (1,5)
- Абзацный отступ (красная строка): 1,25 см
- Выравнивание: по ширине
- Интервал до/после абзаца: 0 пт
- Дополнительное выделение, увеличение интервалов между строками и буквами НЕ допускается

---

## 3. Нумерация страниц

- Арабские цифры, внизу страницы, по центру
- Шрифт нумерации: Times New Roman, 12 пт
- Без точки после номера
- Сквозная нумерация от титульного до последнего листа
- На титульном листе и содержании номер не проставляется (но они входят в общую нумерацию)

---

## 4. Заголовки глав

- Каждая новая глава начинается с новой страницы (разрыв страницы перед заголовком)
- Печатаются ЗАГЛАВНЫМИ буквами
- Выравнивание: по центру
- Без абзацного отступа
- Шрифт: 14 пт, полужирный
- Точка в конце НЕ ставится
- Переносы слов в заголовке НЕ допускаются
- Слово «Глава» и значок параграфа в названии НЕ ставятся (только номер: «1. ТЕОРЕТИЧЕСКИЕ ОСНОВЫ...»)

---

## 5. Заголовки параграфов

- Печатаются строчными буквами (кроме первой)
- Выравнивание: по центру
- Без абзацного отступа
- Шрифт: 14 пт, полужирный
- Точка в конце НЕ ставится
- Нумерация: в пределах главы (1.1., 1.2., 2.1., ...)
- Переносы слов НЕ допускаются

### Отступы вокруг параграфов:

- **Между названием главы и первым параграфом**: ОДИНАРНЫЙ интервал (одна строка с одинарным межстрочным интервалом, НЕ полуторным). Реализация: добавить пустой параграф с межстрочным интервалом 1.0 (одинарный) между заголовком главы и заголовком первого параграфа.
- **Перед названием нового параграфа** (кроме первого в главе): одна пропущенная строка (пустой параграф с полуторным интервалом)
- **После названия параграфа**: НЕ должно быть пропущенных строк. Текст параграфа начинается сразу на следующей строке (без пустого параграфа между заголовком и текстом).

---

## 6. Введение, заключение, список источников

- Начинаются с новой страницы
- Заголовок: заглавными буквами, по центру, полужирный, без абзацного отступа
- Правила оформления текста — как для основного текста

---

## 7. Таблицы

### Расположение надписей:

1. **«Таблица X.X.X»** — в правом верхнем углу (выравнивание по правому краю), без абзацного отступа, шрифт 14 пт, **НЕ жирный**
2. **Название таблицы** — ниже, по центру, без абзацного отступа, шрифт 14 пт, **НЕ жирный** (обычное начертание)
3. Сама таблица
4. **«Источник: ...»** — сразу под таблицей, **БЕЗ пропуска строки**, шрифт 12 пт, выравнивание по ширине, абзацный отступ 1,25 см

### Формат таблицы:

- Шрифт внутри таблицы: Times New Roman, 12 пт
- Межстрочный интервал внутри таблицы: одинарный (1,0)
- Границы: одинарная линия, чёрный цвет
- Выравнивание таблицы: по центру страницы
- Заголовки столбцов: полужирные, выравнивание по центру
- Данные: первый столбец — по левому краю, остальные — по центру
- Цифровые данные: по правому краю, по классам и разрядам

### Нумерация:

- Формат: X.X.X (номер_главы.номер_параграфа.порядковый_номер)
- Пример: Таблица 1.1.1, Таблица 2.1.3

### Отступы вокруг таблицы («таблица должна дышать»):

- **Перед таблицей** (перед надписью «Таблица X.X.X»): пустая строка (пустой параграф с полуторным интервалом)
- **После таблицы** (после строки «Источник: ...»): пустая строка (пустой параграф с полуторным интервалом)
- Надпись «Таблица X.X.X», название таблицы, сама таблица и «Источник:» — это единый блок БЕЗ пустых строк между ними

### Перенос таблицы:

- Если таблица не умещается на одной странице, на следующей в правом верхнем углу: «Продолжение таблицы X.X.X»
- Нельзя оставлять на предыдущей странице только заголовок и шапку

---

## 8. Рисунки (графики, диаграммы, схемы)

### Расположение подписей:

1. Сам рисунок (изображение) — по центру
2. **«Источник: ...»** — сразу под рисунком, **БЕЗ пропуска строки**, шрифт 12 пт, выравнивание по ширине, абзацный отступ 1,25 см
3. **«Рис. X.X.X. Название рисунка»** — сразу после источника, **БЕЗ пропуска строки**, шрифт 14 пт, выравнивание по ширине, абзацный отступ 1,25 см

### Отступы вокруг рисунка («рисунок должен дышать»):

- **Перед рисунком**: пустая строка (пустой параграф с полуторным интервалом)
- **После рисунка** (после строки «Рис. X.X.X. Название»): пустая строка (пустой параграф с полуторным интервалом)
- Рисунок, «Источник:» и «Рис. X.X.X.» — это единый блок БЕЗ пустых строк между ними

### Нумерация:

- Формат: X.X.X (аналогично таблицам, но отдельная нумерация)
- Сокращение: «Рис.» (с точкой)
- После номера — точка, затем пробел и название
- Точка в конце названия НЕ ставится

---

## 9. Формулы

- Выделяются в отдельную строку
- Абзацный отступ 1,25 см (формула начинается с красной строки)
- Номер формулы: в круглых скобках, справа, через табуляцию
- Формат номера: (X.X.X)
- Перед формулой: пустая строка
- После формулы (после пояснений): пустая строка
- Пояснения начинаются со слова «где» (без двоеточия), каждый символ — с новой строки
- Перед пояснениями (после формулы): запятая

---

## 10. Перечисления (списки)

- Выделяются абзацным отступом 1,25 см
- Маркер первого уровня: тире (–, среднее тире) или строчная буква со скобкой: а) б) в)
- Маркер второго уровня: арабские цифры со скобкой: 1) 2) 3), со смещением вправо на два знака
- После каждого пункта: точка с запятой (;)
- После последнего пункта: точка (.)
- Выравнивание: по ширине

---

## 11. Средние тире вместо длинных

**ВАЖНО:** Все длинные тире (—, U+2014) по всему тексту документа необходимо заменить на средние тире (–, U+2013). Это касается:
- текста абзацев
- перечислений
- подписей к таблицам и рисункам
- заголовков
- любого другого текстового контента

Реализация: перед вставкой текста в документ выполнить глобальную замену символа `—` (em dash) на `–` (en dash).

---

## 12. Висящие предлоги (неразрывные пробелы)

**ВАЖНО:** По тексту не должно быть «висящих» предлогов — коротких слов (предлогов, союзов, частиц), оставшихся в конце строки отдельно от следующего слова.

Список слов, после которых нужно ставить неразрывный пробел: «в», «на», «с», «к», «у», «о», «и», «а», «но», «по», «за», «из», «от», «до», «не», «ни», «об», «то», «ко», «во», «со», «же», «бы», «ли», «их», «ей», «ею», «её», «его».

Реализация: заменить обычный пробел после этих коротких слов на **неразрывный пробел** (non-breaking space, `\u00A0`). В python-docx это делается путём замены символа пробела в тексте run'а.

**НЕ использовать** Shift+Enter (мягкий перенос строки / `<w:br/>`), так как это ломает вёрстку при изменении ширины текста. Правильное решение — именно неразрывный пробел.

---

## 13. Вывод по главе

- Заголовок «Вывод по главе X» — по центру, полужирный, без абзацного отступа
- Перед заголовком: пустая строка
- После заголовка: текст начинается сразу (без пустой строки), аналогично параграфу

## 14. Технические детали реализации (Python + python-docx)

### Структура скрипта:

```python
# Основные импорты
from docx import Document
from docx.shared import Pt, Cm, Mm, RGBColor
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.enum.table import WD_TABLE_ALIGNMENT
from docx.oxml.ns import qn, nsdecls
from docx.oxml import parse_xml
```

### Ключевые функции:

1. `setup_document()` — создание документа с полями и стилем Normal
2. `add_page_numbers()` — нумерация страниц (12 пт, по центру, внизу)
3. `add_paragraph()` — универсальная функция добавления параграфа с параметрами
4. `add_empty_line(doc, spacing=1.5)` — добавление пустой строки с заданным интервалом
5. `create_table()` — создание таблицы с границами и форматированием
6. `replace_dashes(text)` — замена длинных тире на средние
7. `fix_hanging_prepositions(text)` — замена пробелов после предлогов на неразрывные
8. `process_markdown()` — основной парсер markdown

### Алгоритм обработки отступов:

```
ГЛАВА (новая страница, заглавные, жирный, по центру)
  ↓ пустой параграф с ОДИНАРНЫМ интервалом (line_spacing=1.0)
Параграф 1.1 (жирный, по центру)
  ↓ БЕЗ пустой строки — текст начинается сразу
Текст параграфа...
  ↓ пустой параграф с полуторным интервалом (line_spacing=1.5)
Параграф 1.2 (жирный, по центру)
  ↓ БЕЗ пустой строки — текст начинается сразу
Текст параграфа...
```

### Алгоритм обработки таблиц:

```
Текст до таблицы...
  ↓ пустой параграф (полуторный интервал) — "таблица дышит"
«Таблица X.X.X» (по правому краю, НЕ жирный)
Название таблицы (по центру, НЕ жирный)
[ТАБЛИЦА]
«Источник: ...» (12 пт, без пропуска строки)
  ↓ пустой параграф (полуторный интервал) — "таблица дышит"
Текст после таблицы...
```

### Алгоритм обработки рисунков:

```
Текст до рисунка...
  ↓ пустой параграф (полуторный интервал) — "рисунок дышит"
[РИСУНОК] (по центру)
«Источник: ...» (12 пт, без пропуска строки)
«Рис. X.X.X. Название» (14 пт, без пропуска строки)
  ↓ пустой параграф (полуторный интервал) — "рисунок дышит"
Текст после рисунка...
```

### Обработка текста перед вставкой:

```python
def process_text(text):
    """Обрабатывает текст перед вставкой в документ."""
    # 1. Замена длинных тире на средние
    text = text.replace('—', '–')
    # 2. Замена пробелов после предлогов на неразрывные
    prepositions = ['в', 'на', 'с', 'к', 'у', 'о', 'и', 'а', 'но', 
                    'по', 'за', 'из', 'от', 'до', 'не', 'ни', 'об',
                    'то', 'ко', 'во', 'со', 'же', 'бы', 'ли',
                    'их', 'ей', 'ею', 'её', 'его']
    for prep in prepositions:
        # Замена в начале строки
        text = re.sub(
            r'(^|\s)(' + prep + r')\s',
            r'\1\2\u00A0',
            text,
            flags=re.IGNORECASE
        )
    return text
```

## 15. Контрольный чек-лист

Перед финальным сохранением документа проверить:

- [ ] Все длинные тире (—) заменены на средние (–)
- [ ] После предлогов стоят неразрывные пробелы
- [ ] Между главой и первым параграфом — одинарный интервал
- [ ] Перед каждым параграфом (кроме первого в главе) — пустая строка
- [ ] После заголовка параграфа — НЕТ пустой строки
- [ ] Названия таблиц — НЕ жирные
- [ ] «Источник:» под таблицами/рисунками — без пропуска строки, 12 пт
- [ ] Перед и после визуальных элементов (таблица/рисунок/схема) — пустая строка
- [ ] Таблицы и рисунки «дышат» — отступы в одну строку от основного текста
- [ ] Шрифт везде Times New Roman
- [ ] Нумерация страниц: 12 пт, по центру, внизу
- [ ] Каждая глава — с новой страницы
- [ ] Параграфы идут друг за другом (без новой страницы)
- [ ] Заголовки глав — ЗАГЛАВНЫЕ, по центру, жирные
- [ ] Заголовки параграфов — строчные (кроме первой), по центру, жирные

## Примечания

- Если в markdown-тексте встречаются блоки кода (```...```), они интерпретируются как заглушки для рисунков/схем — вставляется текст-заполнитель «[Рисунок — см. оригинал]»
- Формат перечислений определяется автоматически по маркеру в markdown: `- ` → тире, `а) ` → буква, `1) ` → цифра
- Все ссылки на источники [X, с. Y] сохраняются как есть в тексте
